# Memory Log - 2026-03-29

## RED-82: Centralize IGNORE_DIRS across scanners (DONE)

**Task:** Consolidate duplicate `IGNORE_DIRS` constants defined in multiple scanner files into a single shared source.

### Implementation

**Created:** `src/scan/ignore.ts`
- `DEFAULT_IGNORE_DIRS` - Centralized list of directories to skip (node_modules, dist, build, .next, .git, target, __pycache__, venv, vendor, test, etc.)
- `DEFAULT_IGNORE_EXTENSIONS` - Centralized list of file extensions to skip (.min.js, .map, .lock, images, binaries, archives, etc.)
- `loadRedwoodIgnore(repoPath)` - Async function to load .redwoodignore from repo root
- `parseIgnoreFile(content)` - Parse .redwoodignore content into glob patterns
- `isIgnored(file, repoPath, patterns)` - Check if a file matches any ignore pattern
- `shouldSkipDir(dirName, dirPath, repoPath, patterns)` - Check if a directory should be skipped entirely

**Updated all scanners to import from `./ignore.js`:**
- `patterns.ts` - imports DEFAULT_IGNORE_DIRS, DEFAULT_IGNORE_EXTENSIONS, loadRedwoodIgnore, isIgnored, shouldSkipDir
- `secrets.ts` - imports DEFAULT_IGNORE_DIRS, DEFAULT_IGNORE_EXTENSIONS, loadRedwoodIgnore, isIgnored, shouldSkipDir
- `mcp.ts` - imports DEFAULT_IGNORE_DIRS, loadRedwoodIgnore, isIgnored, shouldSkipDir
- `agent-chain-validator.ts` - imports DEFAULT_IGNORE_DIRS, loadRedwoodIgnore, shouldSkipDir

**Result:**
- Build succeeds (`npm run build`)
- Scanner runs correctly (`node dist/cli.js scan .`)
- All scanners now use a single source of truth for ignore patterns
- Users can customize ignore patterns via .redwoodignore file
